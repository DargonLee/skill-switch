# SkillSwitch Design Context

> A refined, editorial-technical aesthetic for developer tools. This document captures the design DNA of SkillSwitch to ensure consistent, high-quality implementation across all features.

---

## Product Identity

**SkillSwitch** is a native desktop application for managing AI prompts and skills. Think of it as a "personal prompt library" for developers who work with AI tools.

### Core Metaphors
- **Library** вҖ” Organized collection, not a messy pile
- **Editor** вҖ” Focused writing experience, not a dashboard
- **Tool** вҖ” Functional utility, not a social platform

### User Archetype
- Developer who uses AI assistants daily
- Values efficiency over discoverability
- Prefers keyboard shortcuts to mouse navigation
- Works across multiple projects/contexts

---

## Design Principles

### 1. Editorial Restraint
Less is more. Every element must earn its place. Avoid decorative flourishes.

```
GOOD: Clean card with subtle shadow, clear hierarchy
BAD: Gradient borders, glowing effects, animated icons
```

### 2. Typographic Hierarchy
Typography is the primary design tool. Use weight, size, and spacingвҖ”not colorвҖ”to create hierarchy.

```
GOOD: Bold name, muted secondary text, subtle timestamp
BAD: Colorful badges, multiple accent colors per card
```

### 3. Warm Minimalism
Neutral palette with intentional warmth. The amber accent should feel refined, not playful.

```
GOOD: oklch warm neutrals, single amber accent
BAD: Cool grays, multiple accent colors, neon highlights
```

### 4. Functional Density
Information-rich but not cluttered. Optimize for scanning and quick comprehension.

```
GOOD: 280px sidebar, 4px padding tags, truncated descriptions
BAD: Wide empty margins, oversized buttons, sparse layouts
```

### 5. Native Feel
Behave like a native desktop app, not a web page.

- Use monospace display font (JetBrains Mono)
- Support system dark mode
- Keyboard-first navigation
- Instant interactions (no loading spinners for local data)

## Visual Language

### Color System (OKLCH)

| Token | Light | Dark | Usage |
|-------|-------|------|-------|
| `--canvas` | `oklch(98.5% 0.002 70)` | `oklch(12% 0.008 70)` | Background |
| `--ink` | `oklch(15% 0.02 70)` | `oklch(96% 0.005 70)` | Primary text |
| `--accent` | `oklch(68% 0.18 65)` | `oklch(75% 0.16 65)` | Primary action |
| `--divider` | `oklch(90% 0.005 70)` | `oklch(25% 0.01 70)` | Borders |

### Typography Scale

| Size | Value | Usage |
|------|-------|-------|
| `--text-xs` | 11px | Timestamps, labels |
| `--text-sm` | 13px | Secondary text, tags |
| `--text-base` | 15px | Body text |
| `--text-lg` | 18px | Section headers |
| `--text-xl` | 22px | Page titles |

### Spacing Scale

Base unit: 4px (0.25rem)

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 4px | Inline gaps |
| `--space-2` | 8px | Tag padding |
| `--space-3` | 12px | Card internal padding |
| `--space-4` | 16px | Section gaps |
| `--space-6` | 24px | Card grid gap |

### Border Radius

- `--radius-sm` (4px): Tags, small elements
- `--radius-md` (6px): Buttons, inputs
- `--radius-lg` (10px): Cards, modals

## Interaction Patterns

### Keyboard Navigation

| Shortcut | Action |
|----------|--------|
| `вҢҳK` | Open search modal |
| `Escape` | Close modal / Cancel |
| `Enter` | Submit form / Confirm |
| `вҶҗ` `вҶ’` | Navigate in some contexts |

### Animations

**Use sparingly.** Animations should feel instant, not theatrical.

```css
/* Preferred: Fast, functional */
--transition-fast: 120ms ease-out;
--transition-normal: 200ms ease-out;

/* Avoid: Slow, decorative */
animation: fadeIn 500ms ease-out; /* NO */
```

**Allowed animations:**
- Card hover lift (1px, fast)
- Modal fade in (150ms)
- Sidebar slide (200ms)

**Avoid:**
- Loading spinners (use skeletons or nothing)
- Bounce effects
- Gradient animations
- Particle effects

### Empty States

Simple message + optional action. No illustrations.

```tsx
<div className={s.empty}>
  <p className={s.emptyText}>No skills yet</p>
  <button className={s.emptyAction}>Create your first skill</button>
</div>
```

## Writing Style

### UI Copy

- Concise, action-oriented
- Sentence case (not Title Case)
- No periods on single-line text

```
GOOD: "Save" "Delete" "New skill"
BAD: "Save Changes" "Are you sure you want to delete?"
```

### Timestamps

Use relative time for recent items:

| Time | Format |
|------|--------|
| Today | "today" |
| Yesterday | "yesterday" |
| < 7 days | "3d ago" |
| < 4 weeks | "2w ago" |
| < 12 months | "3mo ago" |
| Older | "Mar 15" |

### Error Messages

Specific and actionable:

```
GOOD: "Name is required"
GOOD: "Skill not found. It may have been deleted."
BAD: "An error occurred"
BAD: "Invalid input"
```

## What to Avoid

вқҢ **No playful elements**
- No confetti, celebrations, or delight moments
- No emoji in UI (user content is fine)
- No illustration-style empty states

вқҢ **No dark patterns**
- No automatic data collection
- No upsell prompts
- No forced onboarding

вқҢ **No web conventions that clash with native**
- No hamburger menus
- No "click here" links
- No toast notifications for routine actions

вқҢ **No visual noise**
- No badges for everything
- No colored status indicators unless critical
- No gradient backgrounds

## When in Doubt

1. **Simplify** вҖ” Remove before adding
2. **Make it faster** вҖ” 200ms is slow, 100ms is better
3. **Use system font** вҖ” JetBrains Mono for display, system for body
4. **Trust the design tokens** вҖ” Don't invent new colors or sizes
5. **Follow existing patterns** вҖ” Look at SkillCard, AppShell as reference
